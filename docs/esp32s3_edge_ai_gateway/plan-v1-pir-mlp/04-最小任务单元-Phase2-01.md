# 最小任务单元 - Phase 2 (01/04)

> **Phase**: WiFi数据服务模块  
> **范围**: 任务 WF-001 ~ WF-010  
> **预计工时**: 5天

---

## 任务 WF-001: WiFi Kconfig配置

**目标**: 启用WiFi功能配置

**执行步骤**:
1. 在prj.conf添加 `CONFIG_WIFI=y`
2. 添加 `CONFIG_NET_L2_ETHERNET=y`
3. 添加 `CONFIG_NET_IPV4=y`
4. 添加WiFi驱动配置
5. 验证配置无冲突

---

## 任务 WF-002: WiFi初始化

**目标**: 初始化WiFi网络接口

**执行步骤**:
1. 调用 `net_if_init()`
2. 获取WiFi网络接口
3. 配置STA模式
4. 注册网络事件回调
5. 验证初始化成功

---

## 任务 WF-003: WiFi连接管理

**目标**: 实现WiFi连接和IP获取

**执行步骤**:
1. 配置SSID和密码
2. 调用连接API
3. 等待连接成功事件
4. 获取DHCP分配的IP
5. 打印IP地址到日志

---

## 任务 WF-004: 自动重连机制

**目标**: 实现断网自动重连

**执行步骤**:
1. 监听网络断开事件
2. 启动重连定时器
3. 实现指数退避重试
4. 最大重试次数限制
5. 记录重连日志

---

## 任务 WF-005: 连接状态查询

**目标**: 提供WiFi状态查询接口

**执行步骤**:
1. 创建 `wifi_get_status()` 函数
2. 返回连接/断开状态
3. 返回信号强度RSSI
4. 返回当前IP地址
5. 添加状态变化回调

---

## 任务 WF-006: SNTP配置

**目标**: 启用SNTP时间同步

**执行步骤**:
1. 添加 `CONFIG_SNTP=y`
2. 配置NTP服务器地址
3. 启用自动同步
4. 设置同步间隔
5. 验证配置生效

---

## 任务 WF-007: SNTP初始化

**目标**: 初始化SNTP客户端

**执行步骤**:
1. 调用 `sntp_init()`
2. 配置多个NTP服务器
3. 设置时区偏移
4. 启用平滑同步
5. 验证初始化成功

---

## 任务 WF-008: 时间同步

**目标**: 同步网络时间到系统

**执行步骤**:
1. 等待SNTP同步完成
2. 获取Unix时间戳
3. 更新系统时钟
4. 记录同步成功日志
5. 处理同步失败情况

---

## 任务 WF-009: 时区处理

**目标**: 实现UTC+8时区转换

**执行步骤**:
1. 定义时区偏移: +8小时
2. 实现 `utc_to_local()` 函数
3. 处理夏令时(可选)
4. 返回本地时间结构
5. 验证时间转换正确

---

## 任务 WF-010: RTC更新

**目标**: 更新系统RTC时间

**执行步骤**:
1. 获取同步后的时间
2. 转换为RTC格式
3. 调用RTC设置API
4. 验证RTC更新成功
5. 添加RTC读取测试
