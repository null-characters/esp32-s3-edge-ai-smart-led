## 任务 HW-011: 帧解析函数

**目标**: 实现UART通信的帧解析

**执行步骤**:
1. 创建 `frame_parse()` 函数
2. 验证帧头: 检查前2字节是否为0xAA55
3. 提取CMD和LEN字段
4. 验证校验和: 重新计算并与帧内校验和对比
5. 验证帧尾: 检查最后2字节是否为0x0D0A
6. 返回解析结果: 成功/失败 + 提取的数据

**验收标准**:
- 正确帧解析成功
- 错误帧(校验和错误)被拒绝

---

## 任务 HW-012: 校验和计算

**目标**: 实现帧校验和算法

**执行步骤**:
1. 创建 `calc_checksum()` 函数
2. 累加范围: CMD + LEN + DATA
3. 计算方法: 所有字节累加，取低8位
4. 返回: 1字节校验值
5. 添加单元测试: 使用已知数据验证结果

**验收标准**:
- 与协议定义的算法一致
- 单元测试通过

## 任务 HW-014: 响应接收处理

**目标**: 处理驱动板的ACK/NAK响应

**执行步骤**:
1. 创建 `receive_response()` 函数
2. 使用UART异步接收
3. 设置接收超时: 500ms
4. 解析响应: 判断ACK(0x00)或NAK(0xFF)
5. 返回状态码: 0=成功, -1=超时, -2=NAK

**验收标准**:
- 正确识别ACK/NAK
- 超时无响应返回错误码

## 附录: 数据结构和算法

### 时段判断伪代码
```
function get_time_period(hour):
    if hour >= 6 and hour < 12:
        return PERIOD_MORNING
    else if hour >= 12 and hour < 18:
        return PERIOD_AFTERNOON
    else:
        return PERIOD_EVENING
```

### 渐变控制伪代码
```
function fade_control(target_temp, target_brightness, duration_ms):
    current = get_current_settings()
    steps = duration_ms / 50
    temp_step = (target_temp - current.temp) / steps
    bright_step = (target_brightness - current.brightness) / steps
    
    for i from 0 to steps:
        current.temp += temp_step
        current.brightness += bright_step
        set_light(current.temp, current.brightness)
        sleep(50ms)
```
