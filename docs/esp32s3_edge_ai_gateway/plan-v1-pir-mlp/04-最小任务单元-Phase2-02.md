# 最小任务单元 - Phase 2 (02/04)

> **Phase**: WiFi数据服务模块  
> **范围**: 任务 WF-011 ~ WF-020  
> **预计工时**: 5天

---

## 任务 WF-011: HTTP配置

**目标**: 启用HTTP客户端功能

**执行步骤**:
1. 添加 `CONFIG_HTTP_CLIENT=y`
2. 配置HTTP缓冲区大小
3. 启用HTTP超时设置
4. 添加TLS支持(可选)
5. 验证配置无冲突

---

## 任务 WF-012: DNS解析

**目标**: 实现域名解析

**执行步骤**:
1. 配置DNS服务器
2. 实现 `dns_resolve()` 函数
3. 解析wttr.in域名
4. 解析sunrise-sunset.org域名
5. 缓存解析结果

---

## 任务 WF-013: HTTP GET实现

**目标**: 实现HTTP GET请求

**执行步骤**:
1. 创建 `http_get()` 函数
2. 构建HTTP请求头
3. 发送请求到服务器
4. 接收响应数据
5. 返回状态码和响应体

---

## 任务 WF-014: 响应缓冲区

**目标**: 分配HTTP响应缓冲区

**执行步骤**:
1. 定义缓冲区大小: 4KB
2. 使用PSRAM分配
3. 实现缓冲区管理
4. 处理缓冲区溢出
5. 添加内存使用监控

---

## 任务 WF-015: 超时处理

**目标**: 实现HTTP请求超时

**执行步骤**:
1. 设置超时时间: 5秒
2. 实现超时检测
3. 超时后断开连接
4. 返回超时错误码
5. 记录超时日志

---

## 任务 WF-016: wttr.in API封装

**目标**: 封装天气API请求

**执行步骤**:
1. 创建 `weather_fetch()` 函数
2. 构建wttr.in请求URL
3. 设置城市参数
4. 调用HTTP GET
5. 返回原始响应文本

---

## 任务 WF-017: 天气数据解析

**目标**: 解析wttr.in响应

**执行步骤**:
1. 分析wttr.in响应格式
2. 实现文本解析函数
3. 提取天气描述
4. 提取温度信息
5. 返回结构化数据

---

## 任务 WF-018: 天气编码转换

**目标**: 天气文本转数值编码

**执行步骤**:
1. 定义编码表: 晴天=0.0, 多云=0.5, 阴天=1.0
2. 实现 `weather_encode()` 函数
3. 模糊匹配天气描述
4. 返回0-1编码值
5. 处理未知天气类型

---

## 任务 WF-019: 天气缓存结构

**目标**: 设计天气数据缓存

**执行步骤**:
1. 定义 `weather_cache_t` 结构体
2. 包含: 编码值、时间戳、有效期
3. 实现缓存读写接口
4. 处理缓存失效
5. 添加缓存状态查询

---

## 任务 WF-020: 定时更新任务

**目标**: 每小时更新天气

**执行步骤**:
1. 创建定时器线程
2. 设置周期: 1小时
3. 检查缓存有效性
4. 过期则重新获取
5. 更新缓存并通知
