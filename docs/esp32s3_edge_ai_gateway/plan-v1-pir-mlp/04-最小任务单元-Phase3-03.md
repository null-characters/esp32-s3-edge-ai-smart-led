# 最小任务单元 - Phase 3 (03/03)

> **Phase**: AI推理系统模块  
> **范围**: 任务 AI-021 ~ AI-030  
> **预计工时**: 5天

---

## 任务 AI-021: 事件结构定义

**目标**: 定义数据记录事件结构

**执行步骤**:
1. 创建 `data_logger.h`
2. 定义 `lighting_event_t` 结构
3. 包含: 时间戳、事件类型、色温、光强、天气、临近度
4. 定义事件类型枚举
5. 添加结构大小常量

---

## 任务 AI-022: PSRAM记录区

**目标**: 分配PSRAM记录缓冲区

**执行步骤**:
1. 计算缓冲区大小
2. 使用 `k_malloc()` 从PSRAM分配
3. 初始化缓冲区
4. 添加内存检查
5. 记录分配地址

---

## 任务 AI-023: 记录写入

**目标**: 实现事件记录写入

**执行步骤**:
1. 创建 `log_event()` 函数
2. 填充事件结构
3. 写入缓冲区当前位置
4. 更新写入指针
5. 返回写入结果

---

## 任务 AI-024: 环形缓冲管理

**目标**: 实现环形缓冲区管理

**执行步骤**:
1. 定义环形缓冲结构
2. 实现写入: 满了覆盖最旧数据
3. 实现读取: 按索引读取
4. 维护头尾指针
5. 添加缓冲区状态查询

---

## 任务 AI-025: 记录查询

**目标**: 实现记录查询接口

**执行步骤**:
1. 创建 `query_events()` 函数
2. 参数: 起始时间、结束时间
3. 遍历缓冲区匹配记录
4. 返回匹配记录数组
5. 添加查询统计

---

## 任务 AI-026: 主循环集成

**目标**: 集成AI推理到主循环

**执行步骤**:
1. 在主程序初始化AI
2. 加载模型
3. 启动推理线程
4. 处理推理结果
5. 控制灯具输出

---

## 任务 AI-027: 内存优化

**目标**: 优化内存使用

**执行步骤**:
1. 测量当前内存使用
2. 优化张量arena大小
3. 检查内存泄漏
4. 确保arena<8KB
5. 记录优化后内存

---

## 任务 AI-028: 推理延迟测试

**目标**: 测量推理延迟

**执行步骤**:
1. 添加时间测量代码
2. 执行100次推理
3. 计算平均延迟
4. 计算最大/最小延迟
5. 验证<5ms要求

---

## 任务 AI-029: 场景测试

**目标**: 各场景调光正确性测试

**执行步骤**:
