# ESP32-S3 Edge AI Smart LED 用户使用手册

> 多模态 AI 智能照明系统 - 语音交互版本

---

## 产品概述

### 功能特性

| 功能 | 说明 |
|------|------|
| **语音控制** | 唤醒词 + 50+ 命令词，无需联网 |
| **自动控制** | 雷达人体检测 + AI 场景识别 |
| **双轨融合** | 语音优先，自动恢复机制 |
| **亮度调节** | 0-100% 无级调光 |
| **色温调节** | 2700K-6500K 智能调色 |
| **场景预设** | 阅读、观影、用餐、休息、活动 |

### 技术规格

| 项目 | 规格 |
|------|------|
| 主控芯片 | ESP32-S3-N16R8 (16MB Flash + 8MB PSRAM) |
| 语音引擎 | ESP-SR WakeNet + MultiNet |
| AI 推理 | TensorFlow Lite Micro (INT8) |
| 响应延迟 | < 1 秒 (端到端) |
| 工作电压 | 5V DC |
| 功耗 | < 5W (待机) / < 20W (满载) |

## 快速开始

### 第一步：烧录固件

```bash
# 连接开发板到电脑 USB
# 进入 prj-v3 目录
cd prj-v3

# 编译固件
idf.py build

# 烧录固件
idf.py -p /dev/ttyUSB0 flash
# 或使用一键烧录脚本
./tools/flash.sh
```

### 第二步：首次配置

设备启动后会自动进入配网模式：

1. 手机搜索 Wi-Fi 热点：`SmartLED-XXXX`
2. 连接热点，打开浏览器访问：`192.168.4.1`
3. 选择家庭 Wi-Fi，输入密码
4. 设备自动连接，语音提示"网络连接成功"

### 第三步：语音唤醒

对着麦克风说出唤醒词：

> **"小白小白"**

设备会播放提示音并进入命令等待状态（5 秒）。

## 自动控制模式

### 工作原理

```
雷达检测 → AI 分析 → 自动调光
    ↓
  有人/无人
    ↓
  静止/运动
    ↓
  场景识别
```

### 场景自动切换

| 人体状态 | 自动场景 | 亮度 |
|---------|---------|------|
| 无人 | 关灯 | 0% |
| 有人静止 | 阅读模式 | 80% |
| 有人移动 | 活动模式 | 60% |

### 环境变化响应

- 光线变暗 → 自动调亮
- 光线变亮 → 自动调暗
- 人离开 > 30 秒 → 灯光渐暗
- 人离开 > 10 分钟 → 自动关灯

## 双轨融合机制

### 语音优先原则

```
语音命令 → 立即生效 → 锁定 2 小时
                         ↓
                    TTL 租约保护
                         ↓
                    自动控制暂停
```

### 自动恢复条件

| 条件 | 说明 |
|------|------|
| TTL 过期 | 语音锁定 2 小时后自动释放 |
| 环境变化 | 人离开 > 10 分钟自动释放 |
| 手动恢复 | "恢复自动"命令立即释放 |
| 设备重启 | 恢复之前状态 |

## 高级配置

### Wi-Fi 重配

如果需要更换 Wi-Fi 网络：

1. 长按 BOOT 按钮 5 秒
2. 设备进入配网模式
3. 按首次配置流程操作

### 语音灵敏度调节

通过串口命令调整：

```
# 设置唤醒灵敏度 (0-100)
wake_threshold 80

# 设置命令阈值 (0-100)
command_threshold 70
```

### 自动模式参数

```
# 设置无人关灯延时 (秒)
auto_off_delay 600

# 设置亮度调节步长 (%)
brightness_step 10
```

## 故障排除

### 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 无法唤醒 | 麦克风未连接 | 检查 INMP441 接线 |
| 识别率低 | 环境噪声大 | 降低灵敏度或安静环境 |
| 无声音输出 | 扬声器问题 | 检查 MAX98357A 接线 |
| Wi-Fi 连接失败 | 密码错误 | 重新配网 |
| 雷达不工作 | UART 未连接 | 检查 LD2410 接线 |

### 恢复出厂设置

```bash
# 擦除所有配置
idf.py erase-flash

# 重新烧录
idf.py flash
```

### 日志查看

```bash
# 查看实时日志
idf.py monitor

# 或使用分析工具
python tools/log_analyzer.py --port /dev/ttyUSB0
```
