# 最小任务单元 - Phase 1 (01/03)

> **Phase**: 基础硬件通信模块  
> **范围**: 任务 HW-001 ~ HW-010  
> **预计工时**: 5天

---

## 任务 HW-001: 创建项目目录结构

**目标**: 建立清晰的Zephyr项目目录结构

**执行步骤**:
1. 创建 `prj/` 目录作为项目根
2. 创建 `prj/src/` 存放源代码
3. 创建 `prj/src/boards/` 存放板级配置
4. 创建 `prj/src/drivers/` 存放驱动代码
5. 创建 `prj/include/` 存放头文件

**验收标准**:
- 目录结构符合Zephyr项目规范
- 可通过 `ls -R prj/` 查看完整结构

---

## 任务 HW-002: 配置west workspace

**目标**: 初始化Zephyr工作区并配置ESP32-S3支持

**执行步骤**:
1. 确认west已安装: `west --version`
2. 初始化workspace: `west init -m <manifest-url>`
3. 更新依赖: `west update`
4. 添加ESP32-S3 HAL: 确认 `modules/hal/espressif` 存在
5. 验证board支持: `west boards | grep esp32s3`

**验收标准**:
- `west build --help` 正常执行
- 可看到 `esp32s3_devkitc` 等板子

---

## 任务 HW-003: 创建board overlay

**目标**: 配置UART2和GPIO4的设备树overlay

**执行步骤**:
1. 在 `prj/src/boards/` 创建 `esp32s3_devkitc.overlay`
2. 添加GPIO4配置: 设置为输入模式，启用内部上拉
3. 添加UART2配置: 设置TX=GPIO17, RX=GPIO18
4. 设置波特率: 115200bps
5. 检查overlay语法: 使用 `west build` 验证无警告

**验收标准**:
- overlay文件无语法错误
- `west build` 编译通过

---

## 任务 HW-004: 配置prj.conf

**目标**: 启用必要的Zephyr功能模块

**执行步骤**:
1. 创建 `prj/prj.conf`
2. 启用GPIO: `CONFIG_GPIO=y`
3. 启用UART: `CONFIG_SERIAL=y`, `CONFIG_UART_ASYNC_API=y`
4. 启用PSRAM: `CONFIG_ESP_SPIRAM=y`
5. 设置日志级别: `CONFIG_LOG_DEFAULT_LEVEL=3`

**验收标准**:
- 配置文件无冲突
- 编译时所有模块正确链接

---

## 任务 HW-005: GPIO初始化

**目标**: 初始化PIR传感器的GPIO引脚

**执行步骤**:
1. 获取GPIO设备: `device_get_binding("GPIO_0")`
2. 配置GPIO4为输入模式
3. 启用内部上拉电阻
4. 验证设备获取成功
5. 添加错误处理: 设备获取失败时打印日志

**验收标准**:
- GPIO设备获取成功
- 可通过逻辑分析仪看到GPIO4默认高电平

---

## 任务 HW-006: GPIO轮询读取

**目标**: 实现PIR传感器状态的轮询读取

**执行步骤**:
1. 创建 `pir.h` 和 `pir.c` 文件
2. 实现 `pir_get_status()` 函数
3. 使用 `gpio_pin_get()` 读取GPIO4电平
4. 返回0(无人)或1(有人)
5. 在main中添加测试循环: 每秒打印一次状态

**验收标准**:
- 函数返回正确的0/1状态
- 有人经过传感器时状态变化可观测

---

## 任务 HW-007: 中断驱动实现(可选)

**目标**: 使用GPIO中断替代轮询(性能优化)

**执行步骤**:
1. 配置GPIO中断: `gpio_pin_interrupt_configure()`
2. 设置触发方式: GPIO_INT_EDGE_BOTH
3. 注册回调函数: `gpio_init_callback()`
4. 添加回调函数: 在中断中更新全局状态
5. 启用中断: `gpio_add_callback()`

**验收标准**:
- 状态变化通过中断触发
- 中断延迟 < 1ms

---

## 任务 HW-008: 软件消抖

**目标**: 消除PIR传感器的信号抖动

**执行步骤**:
1. 定义防抖时间: 50ms
2. 记录上次状态变化时间戳
3. 在状态变化时检查时间差
4. 小于防抖时间则忽略变化
5. 添加测试: 快速挥手测试防抖效果

**验收标准**:
- 单次触发只产生一次状态变化
- 快速连续触发被合并

---

## 任务 HW-009: UART设备初始化

**目标**: 初始化与驱动板通信的UART2

**执行步骤**:
1. 获取UART设备: `device_get_binding("UART_2")`
2. 配置UART参数: 115200, 8N1
3. 设置发送/接收缓冲区
4. 配置流控制: 无硬件流控
5. 验证设备就绪

**验收标准**:
- UART设备获取成功
- 可用示波器观察到TX引脚信号

---

## 任务 HW-010: 帧封装函数

**目标**: 实现UART通信的帧封装

**执行步骤**:
1. 定义帧格式常量: 帧头0xAA55, 帧尾0x0D0A
2. 创建 `frame_build()` 函数
3. 参数: cmd, data[], data_len
4. 构建帧: [帧头][CMD][LEN][DATA][校验和][帧尾]
5. 返回完整帧buffer

**验收标准**:
- 输出帧格式符合协议规范
- 校验和计算正确
