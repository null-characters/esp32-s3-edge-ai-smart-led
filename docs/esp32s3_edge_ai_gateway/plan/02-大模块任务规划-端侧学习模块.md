# 大模块任务规划 - 端侧学习模块

## 模块概述

| 属性 | 说明 |
|------|------|
| 模块名称 | 端侧学习模块 |
| 所属阶段 | Phase 4 |
| 计划周期 | 4周（可选） |
| 核心目标 | 实现端侧模型微调，支持个性化自适应学习 |

## 功能边界

### 包含功能
- AIfES 训练框架集成
- 用户反馈数据收集
- 最后一层微调（Last Layer Fine-tuning）
- 夜间自动训练任务
- OTA 模型更新机制
- 模型版本管理

### 不包含功能
- 全模型训练（计算资源不足）
- 云端训练同步
- 多用户个性化配置
- 实时在线学习

## 依赖关系

```
Phase 1 (雷达接入模块) ──► 雷达特征
       │
Phase 2 (声音分类模块) ──► 声音特征
       │
Phase 3 (场景智能模块) ──► 融合模型
       │
       ▼
Phase 4 (端侧学习模块)
```

## 技术方案选型

### 方案对比

| 方案 | 描述 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| **A** | TFLM 端侧训练 | 官方支持 | API 不成熟，风险高 | ⭐⭐ |
| **B** | AIfES 框架 | 专为 MCU 设计，成熟稳定 | 需额外集成 | ⭐⭐⭐⭐⭐ |
| **C** | 最后一层微调 | 实现简单，风险低 | 适应能力有限 | ⭐⭐⭐⭐ |

**推荐方案**：先采用方案 C（最后一层微调）验证可行性，后续升级到方案 B（AIfES）

## 关键数据结构

### 用户反馈事件
```c
struct feedback_event {
    uint32_t timestamp;       // 反馈时间
    float features[19];       // 当前特征向量
    float ai_output[2];       // AI 输出 (色温, 亮度)
    float user_correction[2]; // 用户修正值
    uint8_t feedback_type;    // 0=旋钮, 1=按钮
};
```

### 训练样本
```c
struct training_sample {
    float features[19];       // 输入特征
    float target[2];          // 目标输出 (用户期望)
};
```

### 模型版本信息
```c
struct model_version {
    uint8_t major;            // 主版本号
    uint8_t minor;            // 次版本号
    uint8_t patch;            // 补丁号
    uint32_t checksum;        // 校验和
    uint32_t train_samples;   // 训练样本数
    uint32_t timestamp;       // 更新时间
};
```

## 训练参数配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 学习率 | 0.001 | 初始学习率 |
| 批大小 | 8 | 受内存限制 |
| 训练轮数 | 10-50 | 早停控制 |
| 最小样本数 | 50 | 触发训练阈值 |
| 验证比例 | 20% | 验证集占比 |

## 内存规划

| 区域 | 大小 | 说明 |
|------|------|------|
| 训练数据缓冲 | 16 KB | 最多 200 个样本 |
| 梯度缓冲 | 4 KB | 反向传播梯度 |
| 优化器状态 | 2 KB | SGD 动量 |
| 训练 arena | 32 KB | AIfES 工作区 |
| **总计** | **54 KB** | 占 PSRAM 0.7% |

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| AIfES 兼容性问题 | 中 | 高 | 先用最后一层微调验证 |
| 训练数据不足 | 高 | 中 | 合成数据增强 |
| 训练导致设备卡顿 | 中 | 高 | 严格限制在夜间执行 |
| 过拟合 | 中 | 中 | 早停 + 正则化 |
| OTA 更新失败 | 低 | 高 | 自动回滚机制 |
