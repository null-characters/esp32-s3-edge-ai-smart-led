## 技术要点

### 三级优先级仲裁

```
优先级（从高到低）：
┌─────────────────────────────────────────────────────────┐
│ 1. 用户本地明确指令（"关灯"）                            │
│    ↓                                                    │
│ 2. ESP-Claw 云端下发的动态 Lua 脚本规则                  │
│    （例："如果下午 3 点且雷达检测有人，色温设为 4000K"）  │
│    ↓                                                    │
│ 3. v3 版本的本地多模态 AI 默认融合算法（兜底）           │
└─────────────────────────────────────────────────────────┘
```

### Lua 脚本示例

```lua
-- 云端下发的动态规则示例
-- 如果下午 3 点且雷达检测有人，色温设为 4000K
function on_time_check()
    local hour = get_current_hour()
    local presence = get_radar_presence()
    if hour == 15 and presence then
        set_lighting(4000, 70)
    end
end
```
