# 开发阶段 - Phase 2: 音频链路优化

> 解决技术风险：带宽黑洞、TTS 算力雪崩

---

## 阶段目标

1. 实现端侧音频压缩，削减上传带宽 90%
2. 实现云端流式 TTS，支持大模型长文本播报
3. 端到端延迟优化至 <5 秒

---

## 技术风险应对

### 风险一：带宽黑洞

| 问题 | 解决方案 | 预期收益 |
|------|---------|---------|
| PCM 原音频 160KB/5s | OPUS 压缩至 10KB | 带宽削减 94% |
| 上传延迟 3-5s | 压缩后 <0.5s | 延迟削减 3s |

### 风险二：TTS 算力雪崩

| 问题 | 解决方案 | 预期收益 |
|------|---------|---------|
| 长文本无法本地合成 | 云端 TTS API | 无算力限制 |
| 本地 TTS 卡顿 | HTTP Stream 边收边播 | 不阻塞主循环 |

**TTS 切换逻辑：**

```c
typedef enum {
    TTS_MODE_LOCAL,      // 本地 TTS
    TTS_MODE_CLOUD,      // 云端 TTS
    TTS_MODE_AUTO,       // 自动选择
} tts_mode_t;

tts_mode_t select_tts_mode(const char* text, bool network_ok) {
    if (!network_ok) {
        return TTS_MODE_LOCAL;  // 降级模式
    }
    
    size_t text_len = strlen(text);
    if (text_len > 50) {
        return TTS_MODE_CLOUD;  // 长文本用云端
    }
    
    return TTS_MODE_LOCAL;  // 短文本用本地
}
```
