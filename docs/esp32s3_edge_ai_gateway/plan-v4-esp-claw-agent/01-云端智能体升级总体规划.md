# 云端智能体升级总体规划（v4）

> 从"交互设备"到"主动智能体（Agent）"的本质跨越

---

## 版本对比

| 维度 | v3 (离线完全态) | v4 (云端智能体) |
|------|----------------|-----------------|
| **核心能力** | 感知+语音交互 | **感知+云端Agent** |
| **响应速度** | 极快 (<200ms) | 较慢 (1-3s) |
| **理解力** | 固定命令词 | **自然语言理解** |
| **灵活性** | 逻辑固定 (C 语言) | **动态脚本生成 (Lua)** |

## 架构设计

### 双层思维模型

```
┌─────────────────────────────────────────────────────────────────────┐
│                        prj-v4 架构                                    │
├─────────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │              第一反应（本地 L1）                               │   │
│  │  雷达+环境 → 多模态融合 → 意图分流路由器 → 本地决策           │   │
│  │                              │                               │   │
│  │                              ▼ 未命中本地指令                 │   │
│  └──────────────────────────────┼──────────────────────────────┘   │
│                                 │                                   │
│  ┌──────────────────────────────▼──────────────────────────────┐   │
│  │              深思熟虑（云端 L2）                               │   │
│  │  ESP-Claw → 云端大模型 → 五维记忆系统 → 动态规则生成         │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  优先级仲裁：用户本地指令 > 云端Lua规则 > 本地多模态AI兜底            │
│                                                                     │
│  优雅降级：网络断开 → 绿灯 → v3本地模式                              │
└─────────────────────────────────────────────────────────────────────┘
```

### 模块划分

| 模块 | 功能 | 框架 |
|------|------|------|
| **ESP-Claw 集成层** | Agent主循环、五维记忆、工具调用 | ESP-Claw |
| **意图分流层** | 本地/云端分流、高频命令处理 | 自研 |
| **端云仲裁层** | 优先级仲裁、Lua脚本执行 | 自研+Lua |
| **网络层** | Wi-Fi管理、HTTPS通信、API封装 | ESP-IDF |
| **降级层** | 优雅降级、网络监控 | 自研 |

## ESP-Claw 核心特性

### 五维记忆系统

| 记忆类型 | 说明 | 示例 |
|---------|------|------|
| Profile | 用户画像 | "用户1：张经理" |
| Preference | 用户偏好 | "张经理喜欢开会时色温冷一点" |
| Fact | 环境事实 | "会议室下午2点光照最强" |
| Rule | 动态规则 | Lua脚本：自动调节策略 |
| Context | 会话上下文 | "正在讨论会议室灯光" |

### 工具调用

| 工具 | 功能 |
|------|------|
| `set_lighting(brightness, color_temp)` | 调节光照 |
| `get_sensor_data()` | 获取传感器数据 |
| `send_notify(message)` | 发送通知 |

## 意图分流路由器

| 场景 | 处理方式 | 响应时间 | Token成本 |
|------|---------|---------|----------|
| 命中本地高频指令 | 直接本地执行 | <200ms | 0 |
| 复杂句式 | 上云处理 | 1-3s | 按实际消耗 |

**收益**：90%本地执行，10%上云

## 技术难点

| 难点 | 解决方案 |
|------|---------|
| 内存压力 | 冷热数据分离、内存复用 |
| 网络稳定性 | 优雅降级到v3本地模式 |
| Token成本 | 意图分流路由器 |
| 网络阻塞 | Watchdog + 异步事件驱动 |
