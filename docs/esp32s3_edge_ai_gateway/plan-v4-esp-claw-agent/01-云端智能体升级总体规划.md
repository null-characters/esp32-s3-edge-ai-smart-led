# 云端智能体升级总体规划（v4）

> 从"交互设备"到"主动智能体（Agent）"的本质跨越

---

## 1. 项目背景与目标

### 1.1 当前状态（v3）

v3 版本已实现完整的离线多模态交互能力：

| 能力 | 实现状态 | 技术方案 |
|------|---------|---------|
| 雷达感知 | ✅ 已实现 | ESP32-S3 ADC + FFT |
| 语音唤醒 | ✅ 已实现 | ESP-SR 唤醒词 |
| 语音识别 | ✅ 已实现 | 本地命令词识别 |
| 语音合成 | ✅ 已实现 | ESP-TTS 离线合成 |
| 多模态融合 | ✅ 已实现 | 手势+语音+雷达 |
| 智能控制 | ✅ 已实现 | 本地 MLP 模型推理 |

### 1.2 核心痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| **交互僵化** | 只能识别固定命令词，无法理解自然语言 | 用户体验差 |
| **规则固定** | C 语言硬编码逻辑，无法动态更新 | 迭代成本高 |
| **无记忆** | 每次交互独立，无法学习用户习惯 | 智能化程度低 |
| **无主动能力** | 只能被动响应，无法主动服务 | 设备价值有限 |

### 1.3 v4 目标

**核心定位**：从"交互设备"升级为"主动智能体"

| 目标 | 描述 | 衡量指标 |
|------|------|---------|
| 自然语言理解 | 支持自然语言指令，无需固定命令词 | 识别准确率 >90% |
| 动态规则 | 云端下发 Lua 脚本，本地执行 | 规则热更新 <5s |
| 五维记忆 | Profile/Preference/Fact/Rule/Context | 记忆召回准确率 >85% |
| 主动服务 | 基于历史行为预测用户需求 | 预测准确率 >70% |
| 优雅降级 | 网络断开时回退 v3 本地模式 | 降级时间 <1s |

## 2. 版本对比

| 维度 | v3 (离线完全态) | v4 (云端智能体) |
|------|----------------|-----------------|
| **核心能力** | 感知+语音交互 | **感知+云端Agent** |
| **响应速度** | 极快 (<200ms) | L1本地快 + L2云端慢 |
| **理解力** | 固定命令词 | **自然语言理解** |
| **灵活性** | 逻辑固定 (C 语言) | **动态脚本生成 (Lua)** |
| **记忆能力** | 无 | **五维记忆系统** |
| **主动能力** | 无 | **预测性服务** |
| **网络依赖** | 完全离线 | 混合模式（可降级） |

### 4.2 工具调用（Tool Calling）

| 工具名称 | 功能描述 | 参数 | 返回值 |
|---------|---------|------|--------|
| `set_lighting` | 调节灯光亮度和色温 | `brightness`: 0-100<br>`color_temp`: 2700-6500K | `success`: bool |
| `get_sensor_data` | 获取传感器数据 | 无 | `radar_distance`: float<br>`ambient_lux`: float<br>`gesture`: string |
| `send_notify` | 发送通知消息 | `message`: string<br>`channel`: "telegram"\|"feishu" | `success`: bool |
| `set_scene` | 切换场景模式 | `scene_name`: string | `success`: bool |
| `get_user_preference` | 获取用户偏好 | `user_id`: string<br>`preference_key`: string | `value`: any |
| `update_memory` | 更新记忆 | `memory_type`: string<br>`key`: string<br>`value`: any | `success`: bool |

### 6.2 Lua 脚本执行环境

```lua
-- Lua 脚本沙箱环境
-- 可用 API:
--   - set_lighting(brightness, color_temp)
--   - get_sensor_data() -> {radar, lux, gesture}
--   - get_user_preference(key) -> value
--   - log(message)
--   - send_notify(message, channel)

-- 示例: 智能光照补偿规则
function auto_lighting_rule()
    local sensor = get_sensor_data()
    local user_pref = get_user_preference("brightness_sensitivity")
    
    -- 根据环境光照自动调节
    if sensor.lux > 600 then
        -- 环境亮，降低灯光
        set_lighting(50 + user_pref.offset, 4500)
    elseif sensor.lux < 200 then
        -- 环境暗，提高灯光
        set_lighting(90 + user_pref.offset, 3500)
    end
    
    -- 有人靠近时提高亮度
    if sensor.radar < 100 then
        set_lighting(100, 4000)
    end
end

-- 注册为定时回调
register_callback("on_sensor_update", auto_lighting_rule)
```

| 内存占用 | <8KB | 脚本运行时内存限制 |
| API 白名单 | 限制 | 只能调用白名单内的 API |
