## 核心角色定位

ESP32-S3 作为**边缘AI照明网关**，承担四大职责：

1. **数据采集** — WiFi获取天气/日落时间，GPIO读取人体传感器
2. **智能决策** — TFLite Micro模型推理，输出色温+光强
3. **设备控制** — UART串口发送调光命令到驱动板
4. **人机接口** — BLE预留接口（暂不实现App，仅保留协议）

## 数据流

```
PIR传感器 ──► GPIO读取 ──► 特征工程 ──┬──► AI推理 ──► 调光决策
                                     │                │
WiFi天气 ◄──► HTTP Client ──► 缓存 ──┤                ├──► UART ──► 驱动板
                                     │                │
NTP时间 ◄──► SNTP ──► RTC ───► 时间特征┘                └──► BLE Notify(预留)
```

## Zephyr 线程模型

| 线程 | 优先级 | 职责 | 栈大小 |
|------|--------|------|--------|
| Main | 低 | 系统初始化、空闲任务 | 1KB |
| WiFi Manager | 高 | WiFi连接管理、HTTP请求 | 4KB |
| PIR Sensor | 高 | GPIO中断处理、人员检测 | 1KB |
| AI Inference | 中 | 定时推理、模型执行 | 8KB |
| UART Control | 高 | 串口命令发送、状态接收 | 2KB |
| BLE Service | 低 | GATT服务（预留） | 4KB |

> AI推理线程栈较大是因为TFLite Micro推理时需要临时工作空间。

## 数据更新策略

| 数据 | 来源 | 更新频率 | 说明 |
|------|------|----------|------|
| 天气 | wttr.in HTTP | 1小时 | 天气条件 + 云量 |
| 日出日落 | sunrise-sunset.org HTTP | 每天1次 | 基于经纬度计算 |
| PIR传感器 | GPIO中断 | 实时 | 有人/无人状态 |
| AI推理 | TFLM | 有人时每30秒 | 平衡响应与功耗 |
| UART命令 | 串口 | 光强变化时 | 避免频繁发送 |

## 电源管理

- PIR传感器中断唤醒（无需Deep Sleep）
- WiFi保持连接（需要实时获取天气）
- AI推理间隔可调，平衡功耗与响应

## 应用场景：会议室智能照明

### 核心逻辑

```
输入特征:
  - hour_sin, hour_cos   → 当前时间周期性编码
  - sunset_proximity     → 日落临近度（日落前渐变增强）
  - weather              → 天气状态（0=晴, 1=阴）
  - presence             → 人员存在（PIR传感器）
  - sunrise_hour         → 日出时间
  - sunset_hour          → 日落时间

输出控制:
  - color_temp: 2700K ~ 6500K (UART发送)
  - brightness: 0% ~ 100% (UART发送)

典型场景:
  - 阴天白天 → 光强增强20%补偿
  - 接近日落 → 光强逐渐增强
  - 无人 → 强制关灯
  - 深夜 → 限制最大亮度50%
```

### AI价值

- **多维度映射**: 时间+天气+人员 → 神经网络天然支持
- **平滑过渡**: 输出连续渐变，无阶梯跳变
- **可学习**: 记录用户手动调整，持续优化模型
