# 03 - 通信协议设计

## 一、UART与驱动板通信协议

### 物理层

| 参数 | 配置 |
|------|------|
| 波特率 | 115200 bps |
| 数据位 | 8 bit |
| 停止位 | 1 bit |
| 校验 | 无 (None) |
| 流控 | 无 |
| 接口 | UART2 (TX=GPIO17, RX=GPIO18) |

### 帧格式

```
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│  帧头    │  命令字  │  数据长度 │   数据    │  校验和  │  帧尾    │
│ 0xAA55   │  1字节   │  1字节   │  N字节   │  1字节   │  0x0D0A  │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
   2字节     1字节      1字节      N字节      1字节      2字节
```

**字段说明**：
- **帧头**: `0xAA 0x55` 固定两个字节，用于帧同步
- **命令字**: 标识命令类型（见下表）
- **数据长度**: 数据域的字节数（0-255）
- **数据**: 命令参数（可变长度，大端序）
- **校验和**: 从命令字到数据域的累加和（低8位）
- **帧尾**: `0x0D 0x0A` (CRLF)，标识帧结束

### 命令定义

| 命令字 | 名称 | 方向 | 数据长度 | 说明 |
|--------|------|------|----------|------|
| 0x01 | CMD_SET_COLOR_TEMP | ESP→驱动板 | 2 | 设置色温 |
| 0x02 | CMD_SET_BRIGHTNESS | ESP→驱动板 | 1 | 设置光强 |
| 0x03 | CMD_SET_BOTH | ESP→驱动板 | 3 | 同时设置色温+光强 |
| 0x10 | CMD_GET_STATUS | ESP→驱动板 | 0 | 查询当前状态 |
| 0x11 | RESP_STATUS | 驱动板→ESP | 5 | 状态响应 |
| 0x20 | CMD_SET_FADE | ESP→驱动板 | 4 | 渐变设置 |
| 0xE0 | RESP_ACK | 驱动板→ESP | 1 | 确认响应 |
| 0xE1 | RESP_NAK | 驱动板→ESP | 1 | 错误响应 |

### 命令详解

#### 0x01 - 设置色温

**请求帧** (ESP → 驱动板):
```
[AA 55] [01] [02] [0A 74] [7D] [0D 0A]
 帧头   命令  长度  数据    校验  帧尾
 
数据: 0x0A74 = 2676 (小端) → 实际色温2676K
```

- 数据: 色温值，16位无符号整数，范围 2700-6500
- 单位: 开尔文 (K)

#### 0x02 - 设置光强

**请求帧**:
```
[AA 55] [02] [01] [50] [53] [0D 0A]
 帧头   命令  长度  数据 校验  帧尾
 
数据: 0x50 = 80 → 光强80%
```

- 数据: 光强百分比，8位无符号整数，范围 0-100

#### 0x03 - 同时设置色温+光强（推荐）

**请求帧**:
```
[AA 55] [03] [03] [10 0E 50] [71] [0D 0A]
 帧头   命令  长度   数据      校验  帧尾
 
数据: 
  - 0x100E (小端反转) = 0x0E10 = 3600 → 色温3600K
  - 0x50 = 80 → 光强80%
```

- 数据格式: `[色温低8位][色温高8位][光强]` (3字节)

#### 0x10 - 查询状态

**请求帧**:
```
[AA 55] [10] [00] [10] [0D 0A]
 帧头   命令  长度(0) 校验  帧尾
```

**响应帧** (驱动板 → ESP):
```
[AA 55] [11] [05] [10 0E 50 01 00] [XX] [0D 0A]
 帧头   命令  长度     数据               校验  帧尾
 
数据解析:
  - [10 0E]: 色温 3600K (小端)
  - [50]: 光强 80%
  - [01]: 灯具状态 (0=关, 1=开)
  - [00]: 预留
```

#### 0x20 - 渐变设置

**请求帧**:
```
[AA 55] [20] [04] [10 0E 50 1E] [XX] [0D 0A]
 帧头   命令  长度    数据          校验  帧尾
 
数据:
  - [10 0E]: 目标色温 3600K
  - [50]: 目标光强 80%
  - [1E]: 渐变时间 30秒 (0x1E = 30)
```

- 渐变时间单位: 秒，范围 0-255（0表示立即执行）

### 校验和计算

```c
uint8_t calc_checksum(uint8_t *data, uint8_t len) {
    uint8_t sum = 0;
    for (uint8_t i = 0; i < len; i++) {
        sum += data[i];
    }
    return sum;
}

// 示例：计算从命令字开始的校验和
uint8_t frame[] = {0x03, 0x03, 0x10, 0x0E, 0x50};
uint8_t checksum = calc_checksum(frame, sizeof(frame));  // = 0x71
```

### 错误响应

**NAK错误码**:
| 错误码 | 含义 |
|--------|------|
| 0x01 | 帧格式错误 |
| 0x02 | 校验和错误 |
| 0x03 | 命令不支持 |
| 0x04 | 参数超出范围 |
| 0x05 | 执行失败 |

**NAK响应示例**:
```
[AA 55] [E1] [01] [04] [E6] [0D 0A]
 帧头   NAK  长度  错误码 校验  帧尾
 
错误码 0x04 = 参数超出范围
```

### 通信流程

```
ESP32-S3                              驱动板MCU
    │                                      │
    ├───── CMD_SET_BOTH ─────────────────>│
    │     [AA 55 03 03 10 0E 50 71 0D 0A]  │
    │                                      │
    │<──────── RESP_ACK ──────────────────┤
    │     [AA 55 E0 01 00 E1 0D 0A]        │
    │                                      │
    │     (或 RESP_NAK 如果出错)            │
    │                                      │
```

### 超时与重试

| 参数 | 值 | 说明 |
|------|-----|------|
| 响应超时 | 500ms | 等待ACK/NAK超时 |
| 重试次数 | 3次 | 超时后重发 |
| 重试间隔 | 100ms | 重试间隔 |

## 二、HTTP协议（WiFi上行）

### 天气API - wttr.in

**请求**：
```
GET https://wttr.in/Beijing?format=%C+%h
```

**响应示例**：
```
Partly cloudy 60
```

**格式说明**：
- `%C` - 天气状况 (Sunny, Partly cloudy, Cloudy, Overcast...)
- `%h` - 湿度百分比

### 日出日落API - sunrise-sunset.org

**请求**：
```
GET https://api.sunrise-sunset.org/json?lat=39.9042&lng=116.4074&formatted=0
```

**响应示例**：
```json
{
  "results": {
    "sunrise": "2024-01-15T06:45:12+00:00",
    "sunset": "2024-01-15T18:30:45+00:00",
    "day_length": 42333
  }
}
```

**解析**：
- 提取 `sunrise` 和 `sunset` 字段
- 转换为本地时间（北京时间UTC+8）
- 计算日出/日落小时数（0-24）

### 数据缓存策略

```c
struct weather_cache {
    char condition[32];       // 天气状况字符串
    float weather_code;       // 编码后: 0=晴, 0.5=多云, 1=阴
    time_t last_update;       // 上次更新时间
    uint32_t update_interval; // 更新间隔(秒)，默认3600
};

struct sun_cache {
    float sunrise_hour;       // 日出时间 (0-24)
    float sunset_hour;        // 日落时间 (0-24)
    time_t last_update;
    uint32_t update_interval; // 更新间隔(秒)，默认86400
};
```

## 三、BLE协议（预留接口）

暂不实现手机App，但保留BLE GATT服务定义供后续扩展。

### GATT Service 定义

```
Service: EdgeAI Lighting (0xFFF0)
├── Char: Device Status      (0xFFF1)  [Read + Notify]
│   └── 当前灯光状态: 色温、光强、有人/无人
├── Char: Control Command    (0xFFF2)  [Write]
│   └── 手动控制: 设置色温、光强
├── Char: AI Prediction      (0xFFF3)  [Read + Notify]
│   └── AI推理结果和当前特征值
├── Char: Weather Data       (0xFFF4)  [Read]
│   └── 当前天气和日落时间
└── Char: System Config      (0xFFF5)  [Write + Read]
    └── 系统配置: AI开关、渐变时间等
```

### 数据格式（CBOR）

**Device Status (0xFFF1)**:
```json
{
  "ts": 1713868800,
  "color_temp": 3600,
  "brightness": 80,
  "presence": true,
  "weather": 0.5
}
```

**Control Command (0xFFF2)**:
```json
{
  "color_temp": 4500,
  "brightness": 60,
  "fade_time": 2
}
```

**AI Prediction (0xFFF3)**:
```json
{
  "ts": 1713868800,
  "features": {
    "hour_sin": 0.5,
    "hour_cos": 0.866,
    "sunset_proximity": 0.75,
    "weather": 0.5,
    "presence": 1
  },
  "prediction": {
    "color_temp": 3600,
    "brightness": 80
  }
}
```

## 四、数据序列化对比

| 协议 | 格式 | 用途 | 选择理由 |
|------|------|------|----------|
| UART | 自定义二进制帧 | ESP↔驱动板 | 轻量、快速、易解析 |
| HTTP | 文本/JSON | ESP↔云服务 | 标准、易调试 |
| BLE | CBOR | ESP↔手机 | 紧凑、Zephyr原生支持 |
