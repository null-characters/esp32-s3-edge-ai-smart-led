# 最小任务单元 - Phase 1: 雷达接入模块

> **Phase**: 雷达接入模块  
> **范围**: 任务 RD-001 ~ RD-024  
> **预计工时**: 2周

---

## 任务 RD-001: LD2410 原理图设计

**目标**: 完成毫米波雷达模块的硬件接口设计

**执行步骤**:
1. 确认 LD2410 接口: UART (TX/RX/VCC/GND)
2. 分配 ESP32-S3 UART1 引脚: TX=GPIO17, RX=GPIO18
3. 设计电平转换电路 (如需要)
4. 添加去耦电容 (100nF + 10uF)
5. 绘制原理图并评审

**验收标准**:
- 原理图通过评审
- 引脚分配与 ESP32-S3 数据手册一致

---

## 任务 RD-002: PCB 改版布局

**目标**: 完成雷达模块的 PCB 布局

**执行步骤**:
1. 确定雷达模块位置 (避免遮挡)
2. 布线 UART 信号线 (等长匹配)
3. 添加测试点 (TX/RX/VCC/GND)
4. 检查 DRC (设计规则检查)
5. 输出 Gerber 文件

**验收标准**:
- DRC 检查通过
- 信号线长度匹配 < 5mm

---

## 任务 RD-003: 硬件焊接调试

**目标**: 完成雷达模块的硬件焊接和基础调试

**执行步骤**:
1. 检查 PCB 焊盘质量
2. 焊接 LD2410 模块
3. 焊接连接器
4. 目视检查焊接质量
5. 上电测试: 测量 VCC 电压 (5V)

**验收标准**:
- 焊接无短路、无虚焊
- 上电后雷达模块指示灯亮

---

## 任务 RD-004: UART1 初始化

**目标**: 配置 ESP32-S3 的 UART1 与 LD2410 通信

**执行步骤**:
1. 在设备树 overlay 中定义 UART1
2. 配置波特率: 256000 (LD2410 默认)
3. 配置数据格式: 8N1
4. 启用 DMA 接收
5. 在 prj.conf 中启用 UART 配置

**验收标准**:
- `west build` 编译通过
- UART1 设备树节点正确生成

---

## 任务 RD-005: DMA 接收配置

**目标**: 配置 UART DMA 实现高效数据接收

**执行步骤**:
1. 分配 DMA 缓冲区: 2 个 256 字节缓冲区
2. 配置 DMA 通道
3. 实现双缓冲切换逻辑
4. 添加接收完成回调
5. 测试 DMA 接收稳定性

**验收标准**:
- DMA 接收无丢包
- 缓冲区切换无数据丢失

---

## 任务 RD-006: 帧解析实现

**目标**: 解析 LD2410 的 UART 数据帧

**执行步骤**:
1. 研究 LD2410 协议文档
2. 定义帧结构体: `ld2410_frame_t`
3. 实现帧头检测: 0xF4 0xF3 0xF2 0xF1
4. 实现帧尾检测: 0xF8 0xF9 0xFA 0xFB
5. 提取数据字段: 距离、速度、能量、目标数

**验收标准**:
- 正确解析 LD2410 输出帧
- 数据字段值在合理范围内

---

## 任务 RD-007: 数据校验

**目标**: 实现帧校验和验证

**执行步骤**:
1. 计算校验和: 累加和低 8 位
2. 与帧内校验字节比较
3. 校验失败时丢弃帧
4. 统计校验失败率
5. 添加日志记录

**验收标准**:
- 校验失败帧被正确丢弃
- 校验失败率 < 1%

---

## 任务 RD-008: 特征结构定义

**目标**: 定义雷达特征数据结构

**执行步骤**:
1. 创建 `radar_features_t` 结构体
2. 定义字段: distance, velocity, energy, target_count
3. 定义扩展字段: motion_amplitude, motion_period
4. 添加归一化说明注释
5. 放入 `include/radar.h`

**验收标准**:
- 结构体定义清晰
- 字段注释完整

---

## 任务 RD-009: 距离特征提取

**目标**: 提取并归一化距离特征

**执行步骤**:
1. 从帧数据获取距离值 (0-8m)
2. 归一化到 [0, 1]: `distance / 8.0f`
3. 处理异常值: 超出范围时使用上一有效值
4. 添加滤波: 滑动平均 (窗口=5)
5. 输出到特征结构体

**验收标准**:
- 归一化值在 [0, 1] 范围内
- 距离变化平滑无跳变

---

## 任务 RD-010: 速度特征提取

**目标**: 提取并归一化速度特征

**执行步骤**:
1. 从帧数据获取速度值 (-5~+5 m/s)
2. 归一化到 [-1, 1]: `velocity / 5.0f`
3. 处理异常值: 超出范围时置 0
4. 添加滤波: 低通滤波
5. 输出到特征结构体

**验收标准**:
- 归一化值在 [-1, 1] 范围内
- 速度方向正确 (正/负)

---

## 任务 RD-011: 能量特征提取

**目标**: 提取并归一化能量特征

**执行步骤**:
1. 从帧数据获取能量值 (SNR)
2. 归一化到 [0, 1]: `energy / 100.0f` (假设最大 100)
3. 处理异常值: 负值时置 0
4. 添加滤波: 指数平滑
5. 输出到特征结构体

**验收标准**:
- 归一化值在 [0, 1] 范围内
- 能量变化反映目标活动强度

---

## 附录: 相关文件模板

### radar.h
```c
#ifndef RADAR_H
#define RADAR_H

#include <stdint.h>
#include <stdbool.h>

typedef struct {
    float distance;          // 归一化距离 [0, 1]
    float velocity;          // 归一化速度 [-1, 1]
    float energy;            // 归一化能量 [0, 1]
    float target_count;      // 目标数量
    float motion_amplitude;  // 运动幅度
    float motion_period;     // 运动周期
    float distance_variance; // 距离方差
    float energy_trend;      // 能量趋势
} radar_features_t;

int radar_init(void);
int radar_read(radar_features_t *features);
int radar_infer(const radar_features_t *features, float *output);

#endif
```

### ld2410_frame.h
```c
#ifndef LD2410_FRAME_H
#define LD2410_FRAME_H

#include <stdint.h>

typedef struct __attribute__((packed)) {
    uint8_t header[4];       // 0xF4 0xF3 0xF2 0xF1
    uint8_t type;            // 帧类型
    uint16_t length;         // 数据长度
    uint8_t target_state;    // 目标状态
    uint8_t target_count;    // 目标数量
    float distance;          // 目标距离
    float velocity;          // 目标速度
    float energy;            // 目标能量
    uint8_t checksum;        // 校验和
    uint8_t tail[4];         // 0xF8 0xF9 0xFA 0xFB
} ld2410_frame_t;

#endif
```
