## 1. 场景概述

基于雷达和麦克风的多模态数据，智能照明系统需要识别以下场景：

| 场景ID | 场景名称 | 描述 | 照明策略 |
|--------|----------|------|----------|
| 0 | 无人 (empty) | 房间无人 | 关灯 |
| 1 | 会议 (meeting) | 多人静坐讨论 | 冷白光，中等亮度 |
| 2 | 专注 (focus) | 单人静坐工作 | 中性光，高亮度 |
| 3 | 汇报 (presentation) | 演讲/展示 | 聚光模式 |
| 4 | 休息 (break) | 短暂休息 | 暖光，低亮度 |
| 5 | 打扫 (cleaning) | 清洁活动 | 冷白光，全亮度 |

| 场景 | 典型声音 | MFCC 特征 |
|------|----------|-----------|
| 无人 | 静音 | 能量低 |
| 会议 | 人声讨论 | 多人声频谱 |
| 专注 | 键盘声/静音 | 周期性敲击 |
| 汇报 | 单人演讲 | 连续人声 |
| 休息 | 轻声交谈/静音 | 低能量 |

## 3. 照明控制策略

### 3.1 场景参数表

```c
typedef struct {
    uint16_t color_temp;    // 色温 (K)
    uint8_t brightness;     // 亮度 (%)
    uint16_t fade_duration;  // 渐变时间 (ms)
} scene_lighting_t;

static const scene_lighting_t scene_profiles[] = {
    [SCENE_EMPTY]       = { 2700,   0,  1000 },  // 关灯
    [SCENE_MEETING]     = { 5000,  70,  2000 },  // 冷白光
    [SCENE_FOCUS]       = { 4500,  85,  2000 },  // 中性光
    [SCENE_PRESENTATION]= { 5500,  90,  3000 },  // 聚光
    [SCENE_BREAK]       = { 3000,  30,  3000 },  // 暖光
    [SCENE_CLEANING]    = { 6000, 100,  1000 },  // 全亮
};
```

### 3.2 场景优先级

1. **打扫模式**: 最高优先级，确保清洁人员有足够照明
2. **会议模式**: 多人场景优先
3. **汇报模式**: 演讲场景
4. **专注模式**: 单人工作
5. **休息模式**: 低优先级
6. **无人模式**: 自动关灯

## 4. 场景切换逻辑

### 4.1 置信度阈值

```c
#define SCENE_CONFIDENCE_THRESHOLD   0.7f  // 最低置信度
#define SCENE_SWITCH_DEBOUNCE_MS      3000  // 切换防抖 3秒
#define SCENE_HISTORY_SIZE            10    // 历史窗口
```

### 4.2 平滑过渡

- 场景切换时使用渐变过渡
- 渐变时间根据场景差异调整：
  - 关灯: 1秒快速
  - 场景切换: 2-3秒平滑
  - 亮度增加: 较快
  - 亮度降低: 较慢（避免突然变暗）

## 6. 异常处理

### 6.1 传感器故障

- 雷达故障: 使用音频 + 时间规则
- 麦克风故障: 使用雷达 + 时间规则
- 双传感器故障: 使用时间规则 + 手动控制

### 6.2 置信度不足

当所有场景置信度 < 0.5 时：
- 保持当前状态
- 等待更多数据
- 5秒后切换到默认场景（专注模式）
